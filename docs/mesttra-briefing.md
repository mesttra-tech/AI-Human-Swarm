# Mesttra Briefing For Contributors

AI-Human-Swarm is hosted under the `mesttra-tech` GitHub organization. This page gives contributors a lightweight, public-safe orientation to Mesttra and how to engage with this project.

## What Mesttra Is

Mesttra is an education and technology initiative focused on developing people, teams, and practical technology capability. Its public website describes work around technology education, mentoring, digital inclusion, impact-company programs, and connection between talent and the job market.

Public source: <https://www.mesttra.com.br/>

## Why Mesttra Hosts This Project

AI-Human-Swarm fits the same practical learning pattern:

- Help people learn current cyber-defense practices by doing.
- Turn fast-moving threat intelligence into repeatable public guidance.
- Use AI as a learning and execution amplifier, while keeping humans accountable for review.
- Give collaborators a safe place to practice research, defensive analysis, documentation, and responsible open-source contribution.

This project is not a private operating workspace. It is a public, defensive, community-maintained project.

## How To Engage

You can engage in four simple ways:

1. Open an issue with a public threat, source, or defensive topic worth tracking.
2. Improve a daily cyber patch with safer commands, clearer verification, or better public sources.
3. Review pull requests for public safety, clarity, source quality, and defensive scope.
4. Ask to become an approved collaborator if you want to help regularly.

Approved collaborator status is invitation-only. It does not grant direct push access to `main`, secret access, or automatic publishing rights.

## What Good Contributions Look Like

Good contributions are:

- Defensive and educational.
- Public-source-backed.
- Easy to read and review.
- Safe to copy and adapt.
- Clear about what was tested and what was not tested.
- Transparent when AI helped draft, research, or review the work.

Good contributions avoid:

- Private data, credentials, logs, local paths, or customer context.
- Offensive payloads, persistence instructions, stealth instructions, or exfiltration guidance.
- Opaque commands, shortened links, or unreviewable downloads.
- Current-threat claims without sources.

## Engagement Path

If you are new:

1. Read [CONTRIBUTING.md](../CONTRIBUTING.md).
2. Read [GOVERNANCE.md](../GOVERNANCE.md).
3. Review the daily patch template in [daily-cyber-patches/TEMPLATE.md](../daily-cyber-patches/TEMPLATE.md).
4. Start with a small issue or documentation pull request.

If you want to become an approved collaborator, use the Portuguese invitation guide in [docs/collaborator-invitation-pt-BR.md](collaborator-invitation-pt-BR.md) or send the same information in English:

- GitHub username.
- Area of interest in security, AI, development, or operations.
- Type of contribution you want to make.
- Confirmation that you understand and accept the project rules.

## Boundary

This repository is public. Keep contributions generic, defensive, and safe for anyone to read. Do not include private company information, non-public operational details, secret values, customer data, or internal-only context.
