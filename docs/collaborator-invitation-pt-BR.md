# Convite Para Colaboradores Aprovados

Olá, pessoal.

Estou abrindo um projeto público no GitHub para criar um fluxo diário de inteligência e prevenção em cibersegurança, com apoio de IA.

A ideia é usar humanos + IA para acompanhar ameaças recentes, vazamentos em instalações de pacotes, riscos em CI/CD, agentes de IA, MCP, extensões, dependências, CVEs exploradas e práticas defensivas que qualquer pessoa possa copiar, revisar e aplicar com segurança.

O repositório é público, mas o papel de colaborador aprovado será apenas por convite. Qualquer pessoa poderá abrir issues ou pull requests, mas nenhuma contribuição será publicada sem revisão manual. O objetivo é manter o projeto aberto, útil e seguro.

## Regras Principais

- Contribuições devem ser defensivas, educacionais e baseadas em fontes públicas.
- Não incluir dados confidenciais, clientes, informações internas, segredos, tokens, logs privados ou caminhos locais.
- Não incluir payloads ofensivos, comandos perigosos, links encurtados, código ofuscado ou instruções de exfiltração.
- Se a contribuição foi criada com ajuda de IA, informe isso no pull request.
- Toda recomendação sobre ameaças atuais deve ter fonte pública confiável sempre que possível.
- Toda mudança será revisada antes de entrar no `main`.
- Ser convidado como colaborador aprovado não autoriza push direto para `main` nem acesso a segredos do repositório.

## Como Participar

Se você quiser participar como colaborador aprovado, responda com:

- Seu usuário do GitHub.
- Sua área de interesse em segurança, IA, desenvolvimento ou operações.
- Que tipo de contribuição pretende fazer.
- Confirmação de que entende e aceita as regras do projeto.

Obrigado por ajudar a construir uma defesa coletiva, aberta e responsável.
