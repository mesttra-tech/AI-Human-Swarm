# Daily Cyber Patch - 2026-05-25

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- Treat package installation, editor extensions, and agent tool configuration as one developer supply-chain perimeter, not separate problems.
- Audit GitHub Actions for unsafe triggers, broad `GITHUB_TOKEN` permissions, and OIDC token issuance before running untrusted pull request code.
- Refresh CISA KEV and patch affected Drupal, Langflow, Trend Micro Apex One, Microsoft Defender, and legacy client-side exposures according to your actual asset inventory.

## Copy/Paste Assessment

Run only the checks that match your environment. These commands should not print secret values.

```bash
# Set this to your local workspace before running checks.
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
cd "$WORKSPACE_ROOT"
```

```bash
# Find package manifests and lockfiles.
find "$WORKSPACE_ROOT" -type f \( \
  -name package.json -o \
  -name package-lock.json -o \
  -name pnpm-lock.yaml -o \
  -name yarn.lock -o \
  -name requirements.txt -o \
  -name pyproject.toml -o \
  -name poetry.lock \
\) -not -path '*/node_modules/*' -not -path '*/.venv/*' -not -path '*/venv/*' -print
```

```bash
# Review install-time package scripts without running them.
rg -n '"(preinstall|install|postinstall|prepare)"[[:space:]]*:' \
  -g 'package.json' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Look for package names and package families called out in recent public supply-chain reporting.
# Treat matches as review prompts, not proof of compromise.
AFFECTED_NPM_PATTERN='(@antv/|echarts|size-sensor|jest-canvas-mock|node-ipc|@tanstack/|mcp-echarts|mcp-mermaid)'

rg -n --hidden "$AFFECTED_NPM_PATTERN" \
  -g 'package.json' \
  -g 'package-lock.json' \
  -g 'pnpm-lock.yaml' \
  -g 'yarn.lock' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Find registry auth configuration files without printing token values.
find "$WORKSPACE_ROOT" -name .npmrc -type f -not -path '*/node_modules/*' -print

rg --files-with-matches --hidden -i '(_authToken|NPM_TOKEN|npm_[A-Za-z0-9]{20,})' \
  -g '.npmrc' \
  -g '!**/.git/**' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: scan workflows for unsafe triggers, broad permissions, OIDC usage, and secret access.
rg -n 'pull_request_target|workflow_run|repository_dispatch|permissions:|id-token:|secrets\\.|GITHUB_TOKEN|NPM_TOKEN|PYPI|TWINE|AWS_|AZURE_|GCP_|KUBECONFIG|VAULT_' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: list workflow token permission declarations for review.
rg -n 'permissions:' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: flag jobs that can request OIDC tokens.
rg -n 'id-token:[[:space:]]*write|audience:|subject:' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# MCP / agent configs: inventory files that can connect model output to tools.
find "$WORKSPACE_ROOT" -maxdepth 6 \( \
  -name '.mcp.json' \
  -o -name 'mcp.json' \
  -o -name '*mcp*.json' \
  -o -path '*/.claude/settings.json' \
\) -print 2>/dev/null | sort
```

```bash
# MCP / agent configs: review non-secret shape only.
# This intentionally avoids printing env blocks.
find "$WORKSPACE_ROOT" -maxdepth 6 \( \
  -name '.mcp.json' \
  -o -name 'mcp.json' \
  -o -name '*mcp*.json' \
\) -print 2>/dev/null | while read -r file; do
  if command -v jq >/dev/null 2>&1; then
    jq -r 'path(.. | objects | select(has("command") or has("args"))) | @json' "$file" 2>/dev/null \
      | sed "s#^#$file #"
  else
    printf '%s\n' "$file"
  fi
done
```

```bash
# Editor extensions: list extension IDs and versions only.
# If the `code` CLI is not installed, review extensions in the editor UI.
code --list-extensions --show-versions 2>/dev/null || true
```

```bash
# Refresh recently added exploited vulnerabilities from CISA KEV.
# Requires curl and jq. Output is CVE metadata only, not local scan data.
SINCE_DATE="$(
  date -u -v-7d +%F 2>/dev/null || date -u -d '7 days ago' +%F
)"

curl -fsSL https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json \
  | jq -r --arg since "$SINCE_DATE" '
      .vulnerabilities[]
      | select(.dateAdded >= $since)
      | [.dateAdded, .cveID, .vendorProject, .product, .vulnerabilityName, .dueDate]
      | @tsv
    '
```

## Copy/Paste Remediation

If a suspicious package, extension, build, or agent tool may have exposed credentials, rotate affected credentials through the proper account or secret-management UI. Prioritize GitHub tokens, npm tokens, PyPI tokens, cloud credentials, CI/CD secrets, Vault tokens, Kubernetes tokens, SSH deploy keys, package-registry tokens, and webhook secrets. Do not paste secret values into issues, pull requests, logs, or chat.

```bash
# Node projects: investigative install mode when install-time malware is suspected.
# This reduces the chance you execute lifecycle scripts while triaging.
npm ci --ignore-scripts
npm audit --audit-level=high
```

```bash
# Node projects: apply non-breaking audit fixes to the lockfile first, then reinstall and test.
npm audit
npm audit fix --package-lock-only
npm ci
npm test
npm audit --audit-level=high
```

```bash
# GitHub Actions: inspect current default workflow token permissions.
# Requires GitHub CLI auth and repository permission.
gh api repos/<OWNER>/<REPO>/actions/permissions/workflow || true
```

```bash
# GitHub Actions: set default workflow token permissions to read-only.
# Run only if you own/admin the repository.
gh api -X PUT repos/<OWNER>/<REPO>/actions/permissions/workflow \
  -f default_workflow_permissions=read \
  -F can_approve_pull_request_reviews=false
```

```bash
# GitHub Actions: review and reduce `id-token: write`.
# Keep it only on trusted jobs that need cloud federation.
rg -n 'id-token:[[:space:]]*write|pull_request_target|pull_request:' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Editor extensions: remove unneeded extensions after review.
# Replace the placeholder with an extension ID from the inventory.
# code --uninstall-extension <publisher.extension>
```

For KEV-listed CVEs, follow vendor guidance and patch timelines. If you cannot patch immediately, reduce exposure until patching is possible: require authentication, restrict admin interfaces, limit network reachability, add monitoring, and confirm backups or rollback paths.
