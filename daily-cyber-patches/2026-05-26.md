# Daily Cyber Patch - 2026-05-26

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- Treat dependency installation and developer tooling as a high-likelihood credential exposure path (CI tokens, registry tokens, cloud keys) and prioritize rotation when you have a credible exposure window.
- Harden GitHub Actions against token/identity misuse: minimize `permissions`, avoid unsafe trigger contexts, and tightly scope OIDC (`id-token`) usage.
- Assume AI-assisted social engineering and agent/tool poisoning are active: enforce human review on tool execution, pin toolchains, and constrain what automated agents can read/write/run.

## Copy/Paste Assessment

Run only the checks that match your environment. These commands should not print secret values.

```bash
# Set this to your local workspace before running checks.
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
cd "$WORKSPACE_ROOT"
```

```bash
# Find package manifests and lockfiles.
find "$WORKSPACE_ROOT" -type f \( \
  -name package.json -o \
  -name package-lock.json -o \
  -name pnpm-lock.yaml -o \
  -name yarn.lock -o \
  -name requirements.txt -o \
  -name pyproject.toml -o \
  -name poetry.lock \
\) -not -path '*/node_modules/*' -not -path '*/.venv/*' -not -path '*/venv/*' -print
```

```bash
# Review install-time package scripts without running them.
rg -n '"(preinstall|install|postinstall|prepare)"[[:space:]]*:' \
  -g 'package.json' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Check for explicitly referenced high-risk supply-chain packages in manifests and lockfiles.
# (This is a triage list; update based on your threat intel sources.)
AFFECTED_NPM_PATTERN='(@antv/|node-ipc|nx-console|echarts|size-sensor)'

rg -n --hidden "$AFFECTED_NPM_PATTERN" \
  -g 'package.json' \
  -g 'package-lock.json' \
  -g 'pnpm-lock.yaml' \
  -g 'yarn.lock' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Find registry auth configuration files without printing token values.
find "$WORKSPACE_ROOT" -name .npmrc -type f -not -path '*/node_modules/*' -print
rg --files-with-matches --hidden -i '(_authToken|NPM_TOKEN|npm_[A-Za-z0-9]{20,})' \
  -g '.npmrc' \
  -g '!**/.git/**' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions workflow triage: highlight common risk factors.
rg -n '(pull_request_target|workflow_run|repository_dispatch|permissions:|id-token:|actions/checkout@|uses: .*@|secrets\\.|GITHUB_TOKEN|ACTIONS_ID_TOKEN_REQUEST_URL)' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Refresh recently added exploited vulnerabilities from CISA KEV (last 14 days).
# Requires: curl, jq
SINCE_DATE="$(
  python3 - <<'PY'
from datetime import datetime, timezone, timedelta
print((datetime.now(timezone.utc) - timedelta(days=14)).date())
PY
)"

curl -fsSL https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json \
  | jq -r --arg since "$SINCE_DATE" '
      .vulnerabilities[]
      | select(.dateAdded >= $since)
      | [.dateAdded, .cveID, .vendorProject, .product, .vulnerabilityName]
      | @tsv
    '
```

## Copy/Paste Remediation

```bash
# Node projects with a lockfile: apply non-breaking audit fixes to the lockfile first.
npm audit
npm audit fix --package-lock-only
npm ci
npm test
npm audit --audit-level=high
```

```bash
# Investigative install mode when install-time malware is suspected.
npm ci --ignore-scripts
npm audit --audit-level=high
```

```bash
# GitHub Actions: ensure the workflow token isn't overly permissive.
# (This returns configuration metadata; it should not return secret values.)
gh api repos/<OWNER>/<REPO>/actions/permissions/workflow || true
```

If a suspected package, extension, runner, or builder compromise may have exposed credentials, rotate affected credentials through the proper account or secret-management UI. Prioritize GitHub tokens, npm tokens, cloud credentials, CI/CD secrets, package-registry tokens, SSH keys, signing keys, and webhook secrets.
