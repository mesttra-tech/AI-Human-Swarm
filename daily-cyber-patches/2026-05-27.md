# Daily Cyber Patch - 2026-05-27

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- Treat dependency installation (npm/PyPI) and developer tooling (extensions, CI runners) as a privileged path: review install-time scripts and rotate credentials if you have a credible exposure window.
- Harden GitHub Actions against token/identity misuse: minimize `permissions`, avoid unsafe triggers, and tightly scope OIDC (`id-token`) usage (including cloud trust-policy conditions).
- Assume AI-assisted social engineering and agent/tool poisoning are active: enforce human review for tool execution, pin toolchains, and constrain what automated agents can read/write/run.

## Copy/Paste Assessment

Run only the checks that match your environment. These commands should not print secret values.

```bash
# Set this to your local workspace before running checks.
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
cd "$WORKSPACE_ROOT"
```

```bash
# Find package manifests and lockfiles.
find "$WORKSPACE_ROOT" -type f \( \
  -name package.json -o \
  -name package-lock.json -o \
  -name pnpm-lock.yaml -o \
  -name yarn.lock -o \
  -name requirements.txt -o \
  -name pyproject.toml \
\) -not -path '*/node_modules/*' -not -path '*/.venv/*' -print
```

```bash
# Review install-time package scripts without running them.
rg -n '"(preinstall|install|postinstall|prepare)"[[:space:]]*:' \
  -g 'package.json' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Find registry auth configuration files without printing token values.
find "$WORKSPACE_ROOT" -name .npmrc -type f -not -path '*/node_modules/*' -print
rg --files-with-matches --hidden -i '(_authToken|NPM_TOKEN|npm_[A-Za-z0-9]{20,})' \
  -g '.npmrc' \
  -g '!**/.git/**' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: quickly inventory risky patterns (untrusted contexts, wide permissions, OIDC).
rg -n '(pull_request_target|workflow_run|repository_dispatch|permissions:|id-token:|secrets\\.)' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Agent/tooling config inventory: identify MCP servers and tool config files for review (paths only).
find "$WORKSPACE_ROOT" -type f \\( -name '*mcp*' -o -name '*agent*' -o -name '*tool*' \\) \
  -not -path '*/node_modules/*' -not -path '*/.venv/*' -not -path '*/.git/*' -print \
  | head -n 200 || true
```

## Copy/Paste Remediation

```bash
# Node projects with a lockfile: apply non-breaking audit fixes to the lockfile first.
npm audit
npm audit fix --package-lock-only
npm ci
npm test
npm audit --audit-level=high
```

```bash
# Investigative install mode when install-time malware is suspected.
npm ci --ignore-scripts
npm audit --audit-level=high
```

```bash
# Python projects: use a dependency vulnerability scanner (choose one your org supports).
# Example (pip-audit):
python3 -m pip install --upgrade pip
python3 -m pip install pip-audit
pip-audit || true
```

If a suspected package, extension, runner, or builder compromise may have exposed credentials, rotate affected credentials through the proper account or secret-management UI. Prioritize GitHub tokens, npm tokens, cloud credentials, CI/CD secrets, package-registry tokens, SSH keys, signing keys, and webhook secrets.
