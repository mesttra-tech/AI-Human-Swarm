# Daily Cyber Patch - 2026-05-23

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- MCP (Model Context Protocol) tool descriptions are a confirmed prompt-injection surface ("tool poisoning") and approval is an event, not a continuous state — a server approved last week can silently change its tool definitions today ("rug pull").
- The widely used `mcp-remote` package carried CVE-2025-6514 (CVSS 9.6, remote code execution via a malicious OAuth `authorization_endpoint`); installations should be on a patched release.
- Treat every locally configured MCP server as a privileged subprocess: inventory, pin, least-privilege, and snapshot it before granting an agent shell, filesystem, or network reach.
- Treat package installation as a credential-theft surface, not only a dependency-vulnerability surface.
- Prioritize npm lifecycle script review, CI/CD token scope reduction, registry token rotation after suspected exposure, and MCP/agent configuration review.
- Check recent exploited-vulnerability additions for internet-facing systems and AI-adjacent tooling before expanding automation.

## Copy/Paste Assessment

Run only the checks that match your environment. None of these commands print API keys, tokens, or environment values — they extract structure only. Do **not** `cat` MCP configs raw; configs commonly hold secrets in their `env` blocks.

```bash
# Reference paths. WORKSPACE_ROOT is your repo or project dir; USER_HOME is where most
# client configs live. Set WORKSPACE_ROOT before running project-scoped checks.
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
export USER_HOME="$HOME"
```

```bash
# Locate MCP client configuration files. We list paths only; we never print contents here.
find "$USER_HOME" -type f \( \
    -name claude_desktop_config.json \
 -o -name '.mcp.json' \
 -o -name 'mcp.json' \
 -o -name 'mcp_config.json' \
 -o -name '.claude.json' \
\) -not -path '*/node_modules/*' -not -path '*/.git/*' -print 2>/dev/null
```

Common per-client locations to spot-check by hand (use `jq` extraction below, do not `cat`):

- `~/Library/Application Support/Claude/claude_desktop_config.json` (Claude Desktop, macOS)
- `%APPDATA%\Claude\claude_desktop_config.json` (Claude Desktop, Windows)
- `~/.claude.json` (Claude Code user scope)
- `~/.cursor/mcp.json` (Cursor user scope)
- `<repo>/.mcp.json` (Claude Code project scope)
- `<repo>/.cursor/mcp.json` (Cursor project scope)
- `<repo>/.vscode/mcp.json` (VS Code project scope)

```bash
# Enumerate configured MCP server names and the command they launch.
# jq prints only the server key and the command/args structure; env values are NOT printed.
find "$USER_HOME" -type f \( \
    -name claude_desktop_config.json -o -name '.mcp.json' -o -name 'mcp.json' \
\) -not -path '*/node_modules/*' 2>/dev/null \
| while read -r cfg; do
    echo "=== $cfg ==="
    jq '
      (.mcpServers // .servers // {})
      | to_entries
      | map({server: .key, command: (.value.command // .value.type), args: (.value.args // [])})
    ' "$cfg" 2>/dev/null || echo "(unparsable or jq not installed)"
  done
```

```bash
# Identify any use of mcp-remote (CVE-2025-6514). Patched versions are 0.1.16 or later.
command -v npm >/dev/null && npm ls -g mcp-remote 2>/dev/null | grep -E 'mcp-remote@' || true

find "$USER_HOME" -type f \( \
    -name claude_desktop_config.json -o -name '.mcp.json' -o -name 'mcp.json' \
\) -not -path '*/node_modules/*' -print0 2>/dev/null \
| xargs -0 grep -l 'mcp-remote' 2>/dev/null || true
```

```bash
# Filesystem-server reach check. Surface any filesystem MCP server with broad roots
# (e.g., $HOME). We extract only the server key and its args (the roots).
find "$USER_HOME" -type f \( \
    -name claude_desktop_config.json -o -name '.mcp.json' -o -name 'mcp.json' \
\) -not -path '*/node_modules/*' 2>/dev/null \
| while read -r cfg; do
    echo "=== $cfg ==="
    jq '
      (.mcpServers // .servers // {})
      | to_entries[]
      | select((.value.args // []) | tostring | test("server-filesystem|@modelcontextprotocol/server-filesystem"))
      | {server: .key, roots: (.value.args // [])}
    ' "$cfg" 2>/dev/null
  done
```

## Copy/Paste Remediation

```bash
# 1. Patch mcp-remote if installed globally. Confirm the resulting version is >= 0.1.16.
npm ls -g mcp-remote 2>/dev/null | grep -q 'mcp-remote@' \
  && npm install -g mcp-remote@latest
```

```bash
# 2. Restrict filesystem MCP server roots. Open the relevant config in your editor and
#    replace overly broad roots with specific project directories:
#
#    Replace:  "args": ["@modelcontextprotocol/server-filesystem", "/Users/<you>"]
#    With:     "args": ["@modelcontextprotocol/server-filesystem", "/Users/<you>/Work/<project>"]
#
#    Then restart the MCP client (Claude Desktop / Cursor / VS Code).
```

```bash
# 3. Remove unused MCP servers. This is a manual edit of the relevant config file.
#    If the config is project-scoped (e.g., .mcp.json in a repo), commit the change
#    in a normal PR so reviewers see what was removed and why.
```

```bash
# 4. Optional: run a public static auditor over your MCP configurations.
#    Invariant Labs publishes the open-source mcp-scan project on GitHub. Review its
#    README before running, and prefer running it without persistent credentials:
#      https://github.com/invariantlabs-ai/mcp-scan
#    Treat any high-severity finding as a blocker until investigated by hand.
```

If you suspect a configured server may have exfiltrated an API key or token (for example, you discover a tool description change that includes instructions to "read and return" credentials), rotate the affected key through the issuing provider's UI. **Never** paste secret values into issues, pull requests, logs, or chat.
# Node projects with a lockfile: apply non-breaking audit fixes to the lockfile first.
npm audit
npm audit fix --package-lock-only
npm ci
npm test
npm audit --audit-level=high
```

```bash
# Investigative install mode when install-time malware is suspected.
npm ci --ignore-scripts
npm audit --audit-level=high
```

```bash
# GitHub Actions: inspect current default workflow token permissions.
gh api repos/<OWNER>/<REPO>/actions/permissions/workflow || true
```

```bash
# GitHub Actions: set default workflow token permissions to read-only.
# Run only if you own/admin the repository.
gh api -X PUT repos/<OWNER>/<REPO>/actions/permissions/workflow \
  -f default_workflow_permissions=read \
  -F can_approve_pull_request_reviews=false
```

```bash
# GitHub Actions: inspect branch protection on the default branch.
gh api repos/<OWNER>/<REPO>/branches/<DEFAULT_BRANCH>/protection || true
```

If a suspected package, build, extension, or runner compromise may have exposed credentials, rotate affected credentials through the proper account or secret-management UI. Prioritize GitHub tokens, npm tokens, cloud credentials, CI/CD secrets, Vault tokens, Kubernetes tokens, package-registry tokens, SSH keys, and webhook secrets.
