# Daily Cyber Patch - 2026-05-23

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- Treat package installation as a credential-theft surface, not only a dependency-vulnerability surface.
- Prioritize npm lifecycle script review, CI/CD token scope reduction, registry token rotation after suspected exposure, and MCP/agent configuration review.
- Check recent exploited-vulnerability additions for internet-facing systems and AI-adjacent tooling before expanding automation.

## Priority Actions

## Copy/Paste Assessment

Run only the checks that match your environment. These commands should not print secret values.

```bash
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
cd "$WORKSPACE_ROOT"
```

```bash
# Find package manifests and lockfiles.
find "$WORKSPACE_ROOT" -type f \( \
  -name package.json -o \
  -name package-lock.json -o \
  -name pnpm-lock.yaml -o \
  -name yarn.lock -o \
  -name requirements.txt -o \
  -name pyproject.toml \
  -name poetry.lock \
\) -not -path '*/node_modules/*' -not -path '*/.venv/*' -not -path '*/venv/*' -print
```

```bash
# Review install-time package scripts without running them.
rg -n '"(preinstall|install|postinstall|prepare)"[[:space:]]*:' \
  -g 'package.json' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Look for recently discussed npm supply-chain indicators in manifests and lockfiles.
AFFECTED_NPM_PATTERN='(@antv/|echarts-for-react|size-sensor|jest-canvas-mock|node-ipc|@tanstack/)'

rg -n --hidden "$AFFECTED_NPM_PATTERN" \
  -g 'package.json' \
  -g 'package-lock.json' \
  -g 'pnpm-lock.yaml' \
  -g 'yarn.lock' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Find registry auth configuration files without printing token values.
find "$WORKSPACE_ROOT" -name .npmrc -type f -not -path '*/node_modules/*' -print

rg --files-with-matches --hidden -i '(_authToken|NPM_TOKEN|npm_[A-Za-z0-9]{20,})' \
  -g '.npmrc' \
  -g '!**/.git/**' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Review GitHub Actions workflows for risky surfaces.
rg -n --hidden '(pull_request_target|permissions:|id-token:|secrets\.|GITHUB_TOKEN|NPM_TOKEN|PYPI|TWINE|AWS_|AZURE_|GCP_|KUBECONFIG|VAULT_)' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Find MCP and agent configuration files before trusting a repository.
find "$WORKSPACE_ROOT" -maxdepth 6 \( \
  -name '.mcp.json' \
  -o -name 'mcp.json' \
  -o -name '*mcp*.json' \
  -o -path '*/.claude/settings.json' \
\) -print 2>/dev/null | sort
```

```bash
# Review agent/MCP configuration surfaces without printing secret values.
rg -n --hidden '(mcpServers|command|args|env|autoApprove|allow|deny|sandbox|dangerously|full-access|skip-approval)' \
  -g '.mcp.json' \
  -g 'mcp.json' \
  -g '*mcp*.json' \
  -g '.claude/settings.json' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Check durabletask-style runtime indicators on local Linux builders or containers.
find /tmp "$HOME/.cache" -maxdepth 3 \( \
  -name 'managed.pyz' \
  -o -name 'rope-*.pyz' \
  -o -name '.sys-update-check' \
  -o -name '.sys-update-check-k8s' \
\) -print 2>/dev/null || true

ps aux 2>/dev/null | rg '(/tmp/managed\.pyz|/tmp/rope-|rope\.pyz)' || true
```

```bash
# Refresh recently added exploited vulnerabilities from CISA KEV.
SINCE_DATE="$(
  python3 - <<'PY'
from datetime import datetime, timezone, timedelta
print((datetime.now(timezone.utc) - timedelta(days=14)).date())
PY
)"

curl -fsSL https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json \
  | jq -r --arg since "$SINCE_DATE" '
      .vulnerabilities[]
      | select(.dateAdded >= $since)
      | [.dateAdded, .cveID, .vendorProject, .product, .vulnerabilityName]
      | @tsv
    '
```

## Copy/Paste Remediation

```bash
# Node projects with a lockfile: apply non-breaking audit fixes to the lockfile first.
npm audit
npm audit fix --package-lock-only
npm ci
npm test
npm audit --audit-level=high
```

```bash
# Investigative install mode when install-time malware is suspected.
npm ci --ignore-scripts
npm audit --audit-level=high
```

```bash
# GitHub Actions: inspect current default workflow token permissions.
gh api repos/<OWNER>/<REPO>/actions/permissions/workflow || true
```

```bash
# GitHub Actions: set default workflow token permissions to read-only.
# Run only if you own/admin the repository.
gh api -X PUT repos/<OWNER>/<REPO>/actions/permissions/workflow \
  -f default_workflow_permissions=read \
  -F can_approve_pull_request_reviews=false
```

```bash
# GitHub Actions: inspect branch protection on the default branch.
gh api repos/<OWNER>/<REPO>/branches/<DEFAULT_BRANCH>/protection || true
```

If a suspected package, build, extension, or runner compromise may have exposed credentials, rotate affected credentials through the proper account or secret-management UI. Prioritize GitHub tokens, npm tokens, cloud credentials, CI/CD secrets, Vault tokens, Kubernetes tokens, package-registry tokens, SSH keys, and webhook secrets.

## Verification

```bash
git status --short
npm test
npm audit --audit-level=high
```

```bash
# Confirm no high-confidence token patterns are present in the current tree.
TOKEN_PATTERN='(github_pat_[A-Za-z0-9_]{20,}|gh[pousr]_[A-Za-z0-9_]{20,}|npm_[A-Za-z0-9]{20,}|sk-[A-Za-z0-9]{20,}|AKIA[0-9A-Z]{16}|-----BEGIN (RSA|DSA|EC|OPENSSH|PRIVATE) KEY-----)'

  -g '!**/.git/**' \

  -g '!**/package-lock.json' \
  -g '!**/pnpm-lock.yaml' \
```

## Not Applicable / Not Tested

- This patch does not prove compromise or clean state; it provides defensive triage and hardening steps.
- CISA KEV checks require `curl` and `jq`.
- npm commands apply to Node.js projects; skip them for repositories without npm manifests.
- `gh api` commands require GitHub CLI authentication and repository permissions.
- Runtime indicator checks are most useful on Linux builders, containers, and CI runners.
