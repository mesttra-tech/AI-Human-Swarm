# Daily Cyber Patch - 2026-05-24

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- Treat package installation and editor extensions as potential credential-theft surfaces; prioritize dependency review and credential rotation after any suspected exposure window.
- Harden GitHub Actions by minimizing `GITHUB_TOKEN` permissions, tightly scoping OIDC (`id-token: write`), and avoiding dangerous triggers such as `pull_request_target` for untrusted code.
- Refresh CISA KEV and patch recently added exploited CVEs (including Drupal Core, Trend Micro Apex One, Langflow, and Microsoft Defender) based on what you actually run.

## Copy/Paste Assessment

Run only the checks that match your environment. These commands should not print secret values.

```bash
# Set this to your local workspace before running checks.
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
cd "$WORKSPACE_ROOT"
```

```bash
# Find package manifests and lockfiles.
find "$WORKSPACE_ROOT" -type f \( \
  -name package.json -o \
  -name package-lock.json -o \
  -name pnpm-lock.yaml -o \
  -name yarn.lock -o \
  -name requirements.txt -o \
  -name pyproject.toml -o \
  -name poetry.lock \
\) -not -path '*/node_modules/*' -not -path '*/.venv/*' -not -path '*/venv/*' -print
```

```bash
# Review install-time package scripts without running them.
rg -n '"(preinstall|install|postinstall|prepare)"[[:space:]]*:' \
  -g 'package.json' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Look for recently discussed npm supply-chain indicators in manifests and lockfiles.
AFFECTED_NPM_PATTERN='(@antv/|echarts|size-sensor|jest-canvas-mock|node-ipc|@tanstack/)'

rg -n --hidden "$AFFECTED_NPM_PATTERN" \
  -g 'package.json' \
  -g 'package-lock.json' \
  -g 'pnpm-lock.yaml' \
  -g 'yarn.lock' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# Find registry auth configuration files without printing token values.
find "$WORKSPACE_ROOT" -name .npmrc -type f -not -path '*/node_modules/*' -print
rg --files-with-matches --hidden -i '(_authToken|NPM_TOKEN|npm_[A-Za-z0-9]{20,})' \
  -g '.npmrc' \
  -g '!**/.git/**' \
  -g '!**/node_modules/**' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: scan workflows for unsafe triggers, OIDC usage, and broad permissions.
rg -n 'pull_request_target|workflow_run|repository_dispatch|permissions:|id-token:|secrets\\.|GITHUB_TOKEN' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: flag jobs that mint OIDC tokens (id-token: write) and verify they do not run for untrusted PRs.
rg -n 'id-token:[[:space:]]*write|pull_request_target|pull_request:' \
  -g '.github/workflows/*.yml' \
  -g '.github/workflows/*.yaml' \
  "$WORKSPACE_ROOT" || true
```

```bash
# MCP / agent configs: inventory files that can connect model output to tools.
find "$WORKSPACE_ROOT" -maxdepth 6 \( \
  -name '.mcp.json' \
  -o -name 'mcp.json' \
  -o -name '*mcp*.json' \
  -o -path '*/.claude/settings.json' \
\) -print 2>/dev/null | sort
```

```bash
# Editor extensions: list VS Code extensions (IDs + versions only).
# If `code` is not installed, review extensions in your editor UI.
code --list-extensions --show-versions 2>/dev/null || true
```

```bash
# Python runtime indicators discussed in durabletask-style incidents.

find /tmp "$HOME/.cache" -maxdepth 3 \( \
  -name 'managed.pyz' \
  -o -name 'rope-*.pyz' \
  -o -name '.sys-update-check' \
  -o -name '.sys-update-check-k8s' \
\) -print 2>/dev/null || true
```

```bash
# Refresh recent exploited-vulnerability additions from CISA KEV (last 7 days).
python3 - <<'PY'
import json, urllib.request, datetime
url='https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json'
with urllib.request.urlopen(url, timeout=30) as r:
    data=json.load(r)
today=datetime.date.today()
cutoff=today-datetime.timedelta(days=7)
for v in sorted(data.get('vulnerabilities', []), key=lambda x: x.get('dateAdded',''), reverse=True):
    da=v.get('dateAdded')
    try:
        d=datetime.date.fromisoformat(da)
    except Exception:
        continue
    if d < cutoff:
        break
    print(f"{da}\t{v.get('cveID')}\t{v.get('vendorProject')}\t{v.get('product')}\t{v.get('vulnerabilityName')}")
PY
```

## Copy/Paste Remediation

If a suspected compromise may have exposed credentials, rotate affected credentials through the proper account or secret-management UI. Do not paste secret values into issues, pull requests, logs, or chat.

```bash
# Node projects: investigative install mode when install-time malware is suspected.
# This reduces the chance you execute lifecycle scripts while you triage.
npm ci --ignore-scripts
npm audit --audit-level=high
```

```bash
# Node projects: apply non-breaking audit fixes to the lockfile first, then reinstall and test.
npm audit
npm audit fix --package-lock-only
npm ci
npm test
npm audit --audit-level=high
```

```bash
# GitHub Actions: locate and review any `pull_request_target` workflows and jobs that request `id-token: write`.
# Treat these as high-risk surfaces for untrusted contributions.
rg -n 'pull_request_target|id-token:[[:space:]]*write' -g '.github/workflows/*.{yml,yaml}' "$WORKSPACE_ROOT" || true
```

```bash
# GitHub Actions: reduce blast radius by explicitly setting least-privilege permissions per workflow/job.
# (Edit YAML; verify with the grep above.)
rg -n 'permissions:' -g '.github/workflows/*.{yml,yaml}' "$WORKSPACE_ROOT" || true
```

```bash
# Editor extensions: remove anything you do not need, and prefer an allowlist process for new installs.
# Verify the current inventory again after cleanup.
code --list-extensions --show-versions 2>/dev/null || true
```

For KEV-listed CVEs, follow vendor guidance and patch timelines. If you cannot patch immediately, reduce exposure (limit network access, require authentication, restrict admin interfaces, and add monitoring) until the patch can be applied.
