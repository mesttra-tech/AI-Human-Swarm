# Daily Cyber Patch - 2026-05-24

Mode: Public defensive guidance. No private environment data included.

## Executive Summary

- An AI coding agent runs in your shell and inherits **every** environment variable in scope — every API token, every cloud credential, every database URL. Most agent tools can read their own env and return it via tool calls, logs, or transcripts.
- `.env` files committed to git — even files deleted in a later commit — leave their contents in the repo's pack files. The right defensive posture treats "ever committed" as equivalent to "leaked."
- Long-lived GitHub Actions secrets are a top CI supply-chain risk; OpenID Connect (OIDC) federation with cloud providers replaces them with short-lived role-chained tokens that expire automatically.

## Threats Reviewed

| Threat | Ecosystem | Why It Matters | Confidence | Sources |
|---|---|---|---|---|
| Agent process inherits parent shell env | Local AI coding agents (Claude Code, Cursor, Aider, etc.) | Agents and their MCP/tool children read whatever was in the parent process; tool outputs can echo env values into transcripts | High | Docker AI Agent Horror Stories; Acuvity "Secrets in the Wind" |
| `.env*` files committed in git history | Any repo using dotenv | Even after deletion + force-push, the values stay in pack files and any historical clone/fork; treat as permanent leak | High | GitHub docs on removing sensitive data; standard secret-hygiene literature |
| Long-lived GitHub Actions secrets | GitHub Actions → AWS / GCP / Azure | Static credentials don't expire; a leaked workflow log, fork, or compromised action retains access until manual rotation | High | GitHub OIDC docs; multiple migration guides |
| direnv auto-loading project secrets | direnv-enabled shells | `cd` into an `allow`-listed directory injects every `.envrc` var into the current shell — including the shell that spawns the agent | Medium | direnv project security docs |
| Shell-history secret exposure | `~/.bash_history`, `~/.zsh_history`, fish history | Any tool with read access to the user's home directory can grep for `export.*=` lines and recover the literal values | Medium | Acuvity "Secrets in the Wind"; standard hardening guides |
| Tool args / agent transcripts capturing secrets | Provider-side conversation logs, Langfuse / OpenInference traces, MCP debug logs | A tool that takes a secret as an argument or echoes it puts it into a transcript stored outside your control | Medium | Acuvity; arXiv:2510.05159 ("Malice in Agentland") |

## Copy/Paste Assessment

None of these commands print the value of any environment variable, the contents of a `.env` file, the contents of a shell-history line, or any token. They report **names**, **paths**, **commits**, and **line counts** only. Do not modify them to print values.

```bash
# Reference paths.
export WORKSPACE_ROOT="<WORKSPACE_ROOT>"
export USER_HOME="$HOME"
```

```bash
# Env-name probe. List env-var NAMES in the current shell that match common
# secret-naming patterns. Values are never printed.
printenv | cut -d= -f1 \
  | grep -Ei '(_TOKEN|_KEY|_SECRET|_PASSWORD|_PASS$|_PWD$|_CREDENTIAL|_API|_AUTH|_SESSION|^AWS_|^GCP_|^AZURE_|^GH_|^GITHUB_|^OPENAI_|^ANTHROPIC_)' \
  | sort -u
```

```bash
# Locate .env-style files under the workspace. Paths only.
find "$WORKSPACE_ROOT" -type f \( \
    -name '.env' -o -name '.env.*' -o -name '*.envrc' \
\) -not -path '*/node_modules/*' -not -path '*/.git/*' -print 2>/dev/null
```

```bash
# Per-repo dotenv leak check. Lists repos that currently track .env files
# (operationally risky) and repos that have EVER committed one in history
# (treat as permanent leak; rotation required).
find "$WORKSPACE_ROOT" -type d -name '.git' -prune 2>/dev/null \
| while read -r gitdir; do
    repo="${gitdir%/.git}"
    tracked=$(git -C "$repo" ls-files -- '.env' '.env.*' 2>/dev/null)
    historic=$(git -C "$repo" log --oneline -- '.env' '.env.*' 2>/dev/null)
    if [ -n "$tracked" ] || [ -n "$historic" ]; then
      echo "=== $repo ==="
      [ -n "$tracked" ]  && echo "currently tracked: $tracked"
      [ -n "$historic" ] && echo "history (commits that touched dotenv):"
      [ -n "$historic" ] && echo "$historic" | head -20
    fi
  done
```

```bash
# direnv status. Shows which directories the current shell has authorized
# to inject env vars. Revoke any you don't actively use.
command -v direnv >/dev/null && direnv status 2>/dev/null || true
ls -1 "$USER_HOME/.local/share/direnv/allow" 2>/dev/null \
  || ls -1 "$USER_HOME/.config/direnv/allow"  2>/dev/null \
  || true
```

```bash
# Shell-history secret-exporting line COUNTS only (never the lines themselves).
# Each number is "how many lines in this history file export something that
# looks like a secret." A non-zero count means rotate + scrub.
for hist in "$USER_HOME/.bash_history" "$USER_HOME/.zsh_history" "$USER_HOME/.local/share/fish/fish_history"; do
  [ -f "$hist" ] || continue
  count=$(grep -cE '(^|[[:space:];|&])(export|setenv|set -x)[[:space:]]+[A-Z_][A-Z0-9_]*=' "$hist" 2>/dev/null || echo 0)
  echo "$hist: $count secret-exporting lines"
done
```

```bash
# GitHub Actions: scan workflow files for static cloud credential patterns.
# Reports file path + line number only (no values printed).
find "$WORKSPACE_ROOT" -type d -name '.github' -prune 2>/dev/null \
| while read -r ghdir; do
    wfdir="$ghdir/workflows"
    [ -d "$wfdir" ] || continue
    grep -nE '(aws-access-key-id|aws-secret-access-key|AWS_ACCESS_KEY_ID|AWS_SECRET_ACCESS_KEY|GOOGLE_CREDENTIALS|AZURE_CREDENTIALS|service_account_key)' \
      "$wfdir"/*.yml "$wfdir"/*.yaml 2>/dev/null | cut -d: -f1-2
  done
```

## Copy/Paste Remediation

```bash
# 1. Move a workstation token to the OS keychain instead of an rc-file export.
#    macOS (Keychain):
#       security add-generic-password -a "$USER" -s "GITHUB_PAT" -w
#       # The -w with no value prompts; the value is not echoed to history or screen.
#       # Then at runtime, look it up via a wrapper:
#       export GITHUB_PAT="$(security find-generic-password -a "$USER" -s "GITHUB_PAT" -w)"
#
#    Linux (libsecret / secret-tool):
#       secret-tool store --label="GITHUB_PAT" service github user "$USER"
#       export GITHUB_PAT="$(secret-tool lookup service github user "$USER")"
#
#    Windows (Credential Manager via PowerShell + SecretManagement module):
#       # See the PowerShell SecretManagement docs; install Microsoft.PowerShell.SecretStore
#       # and use Set-Secret / Get-Secret. Avoid plaintext in PROFILE.ps1.
```

```bash
# 2. Spawn the agent with a SCRUBBED environment, allowlisting only what it needs.
#    Pattern: env -i preserves nothing; you then pass only the explicit vars required.
#    Replace the trailing command with however you launch your agent.
env -i \
  HOME="$HOME" \
  PATH="$PATH" \
  TERM="$TERM" \
  LANG="$LANG" \
  GITHUB_PAT="$(security find-generic-password -a "$USER" -s "GITHUB_PAT" -w)" \
  claude
# Variants of this pattern exist for any agent that respects the parent env.
```

```bash
# 3. Revoke unused direnv authorizations.
#    direnv keeps an allow-list of directories it will auto-load from.
#    Review and revoke entries you no longer use:
direnv status
# For each directory you no longer use:
#   direnv deny <path>
# Removing the .envrc file from the directory also stops auto-load.
```

```bash
# 4. Scrub a `.env` file from a repo's history (DESTRUCTIVE, force-push required).
#    Confirm a backup exists before running this on a shared repo. Coordinate with
#    collaborators because their clones will need to be rebased or re-cloned.
#    GitHub's documented tool for this is git-filter-repo:
#       https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/removing-sensitive-data-from-a-repository
#    The mere existence of the value in any historical commit means the value
#    must be ROTATED at the issuing provider before you finish — purging history
#    does not retract values that were already cloned, forked, or indexed.
```

```bash
# 5. Migrate a GitHub Actions workflow from static AWS keys to OIDC.
#    This is a workflow + cloud-side IAM change, not a one-liner. Outline:
#
#      a) In AWS, register github.com as an OIDC identity provider:
#         https://docs.github.com/en/actions/deployment/security-hardening-your-deployments/configuring-openid-connect-in-amazon-web-services
#      b) Create an IAM role with a trust policy scoped to the SPECIFIC repo
#         (and ideally a specific branch / environment) via the `sub` claim.
#         A wildcard sub claim like "repo:org/*:*" is an anti-pattern.
#      c) Update the workflow:
#           permissions:
#             id-token: write
#             contents: read
#           steps:
#             - uses: aws-actions/configure-aws-credentials@v4
#               with:
#                 role-to-assume: arn:aws:iam::<account>:role/<role-name>
#                 aws-region: <region>
#      d) Verify the workflow passes against the new role on a feature branch.
#      e) Delete the now-unused GH Actions secrets (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
#         AND deactivate the IAM access key in the AWS console.
#
#    GCP and Azure have analogous Workload Identity Federation flows; the same
#    structural pattern applies: identity-provider trust, scoped role, no secrets.
```

```bash
# 6. Shell-history hygiene going forward.
#    Read secrets without echoing them and without leaving them in history.
#    Bash / Zsh:
#       read -rsp "Token: " FOO && export FOO        # value is not echoed
#    With ignorespace enabled, prefixing any command with a leading space
#    keeps it out of the history file:
#       HISTCONTROL=ignorespace      # add to ~/.bashrc
#       setopt HIST_IGNORE_SPACE     # add to ~/.zshrc
#    Then any one-off `  export FOO=...` (leading space) stays out of history.
```

If a leaked or possibly-leaked secret is identified, rotate it at the issuing provider before doing anything else. Rotation is the only step that retroactively reduces exposure — purging history and revoking direnv entries are forward-only.

```bash
# Re-run the env-name probe inside the agent's launching shell. After scrubbing,
# the only secret-named vars should be the ones you explicitly allowlisted.
printenv | cut -d= -f1 \
  | grep -Ei '(_TOKEN|_KEY|_SECRET|_PASSWORD|_PASS$|_PWD$|_CREDENTIAL|_API|_AUTH|^AWS_|^GCP_|^AZURE_|^OPENAI_|^ANTHROPIC_)' \
  | sort -u
```

```bash
# Re-run the per-repo dotenv probe. After scrubbing + rotation, the tracked
# list should be empty. The historic list may still be non-empty (rewriting
# history may not be possible on shared repos) but every secret that was in
# those commits must now be rotated.
```

```bash
# Re-run the workflow scan. After OIDC migration, the workflow files should
# contain no static credential references and should declare `id-token: write`
# permissions on the jobs that authenticate to a cloud provider.
find "$WORKSPACE_ROOT" -type d -name '.github' -prune 2>/dev/null \
| while read -r ghdir; do
    wfdir="$ghdir/workflows"
    [ -d "$wfdir" ] || continue
    grep -nE 'id-token:[[:space:]]+write' "$wfdir"/*.yml "$wfdir"/*.yaml 2>/dev/null
  done
```

```bash
# Re-run the shell-history probe. After scrubbing, secret-exporting line
# counts in each history file should be 0.
```
